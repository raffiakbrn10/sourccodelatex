% --- 1. Pendahuluan ---
\section{Pendahuluan}
\label{sec:pendahuluan}
Pada praktikum minggu ini, modul yang dipelajari berfokus pada sistem \textit{build} menggunakan GCC (\textit{GNU Compiler Collection})
serta penerapan otomatisasi proses kompilasi dengan menggunakan \textit{Makefile}. GCC merupakan salah satu compiler yang banyak digunakan
dalam pengembangan perangkat lunak, khususnya pada sistem operasi berbasis Linux, karena kemampuannya dalam menangani berbagai tahapan kompilasi
seperti preprocessing, compilation, assembly, hingga linking.

Dalam pengembangan perangkat lunak, proses mengubah \textit{source code} menjadi program \textit{executable} merupakan tahapan yang sangat penting
dan mendasar. Proses ini tidak hanya melibatkan penerjemahan kode program ke dalam bahasa mesin, tetapi juga mencakup pengelolaan dependensi antar
file serta pengaturan alur kompilasi agar program dapat dibangun secara efisien dan terstruktur.

Selain itu, penggunaan \textit{Makefile} dalam praktikum ini bertujuan untuk mengotomatisasi proses \textit{build}, sehingga proses kompilasi tidak
perlu dilakukan secara manual setiap kali terjadi perubahan pada kode sumber. Dengan adanya \textit{Makefile}, proses kompilasi menjadi lebih
terorganisir, efisien, dan dapat meminimalkan kesalahan, terutama pada proyek yang memiliki banyak file sumber. Hal ini juga membantu meningkatkan
pemahaman mahasiswa dalam mengelola proyek pemrograman secara lebih profesional di lingkungan Linux.

\subsection{Latar Belakang}
\label{subsec:latar-belakang}
Pemahaman mengenai sistem \textit{build} tidak hanya terbatas pada proses menjalankan perintah \textit{compile}, tetapi juga mencakup pemahaman terhadap
tahapan-tahapan yang terjadi di dalamnya, yaitu \textit{preprocessing}, \textit{compiling}, \textit{assembling}, dan \textit{linking}.
GCC (\textit{GNU Compiler Collection}) merupakan salah satu compiler utama yang umum digunakan dalam pemrograman C/C++ yang mendukung seluruh rangkaian
proses tersebut dari awal hingga menghasilkan program \textit{executable}.

Namun, seiring meningkatnya kompleksitas proyek perangkat lunak serta bertambahnya jumlah file sumber, proses kompilasi secara manual menggunakan perintah
GCC menjadi kurang efisien. Hal ini disebabkan oleh sifatnya yang repetitif, memakan waktu, serta berpotensi menimbulkan kesalahan dalam pengelolaan file yang saling bergantung.

Oleh karena itu, diperlukan sebuah alat otomasi \textit{build} seperti GNU Make. Dengan memanfaatkan \texttt{Makefile}, sistem dapat melakukan pelacakan dependensi
(\textit{dependency tracking}) secara otomatis dan hanya melakukan kompilasi ulang pada bagian kode yang mengalami perubahan. Kombinasi antara pemahaman dasar GCC dan
penguasaan penggunaan \texttt{Makefile} dapat meningkatkan efisiensi waktu serta membantu dalam pengelolaan proyek perangkat lunak yang lebih terstruktur dan profesional.

\subsection{Tujuan}
\label{subsec:tujuan}
\begin{enumerate}
    \item \textit{Memahami proses kompilasi kode sumber C/C++ menggunakan GNU Compiler Collection (GCC).}
    \item \textit{Mengenal dan mempraktikkan tahapan dalam proses build: preparasi, kompilasi, perakitan (assembly), dan linking.}
    \item \textit{Memahami konsep dasar otomatisasi kompilasi (build automation) perangkat lunak.}
    \item \textit{Memahami dan menjelaskan struktur dasar file Makefile (Target, Komponen Syarat, dan Resep).}
    \item \textit{Menggunakan variabel kompilator dan flags penanda dalam Makefile.}
    \item \textit{Menggunakan variabel otomatis (automatic variables) seperti} \verb|$@| \textit{dan} \verb|$^| \textit{untuk efisiensi skrip build.}
\end{enumerate}

\section{Dasar Teori}
\label{sec:dasar-teori}

\subsection{GNU Compiler Collection (GCC)}
\label{subsec:teori-gcc}
Sistem \textit{build} berfungsi untuk mengubah \textit{source code} menjadi file \textit{executable}. Menggunakan GCC (\textit{GNU Compiler Collection}),
proses kompilasi berjalan melalui empat tahapan utama secara berurutan. Pertama adalah \textit{preprocessing} (menangani direktif seperti \verb|#include|
dan \verb|#define|), kedua adalah \textit{compilation} (menerjemahkan kode ke bahasa assembly), ketiga adalah \textit{assembly} (menghasilkan object file
biner/kode mesin), dan terakhir adalah \textit{linking} (menghubungkan object file dengan library eksternal/internal untuk menghasilkan \textit{executable}) \cite{GCCDocumentation, GeeksforGeeksCompilation}.

\subsection{Otomatisasi dengan Makefile}
\label{subsec:teori-makefile}
Untuk mengotomatisasi tahapan ini pada proyek berskala besar, digunakan utilitas GNU Make dengan file konfigurasi bernama \texttt{Makefile}.
GNU Make membaca aturan yang mendefinisikan \textit{target} (file output), \textit{dependency} (file sumber yang dibutuhkan), dan \textit{recipe} (perintah eksekusi).
Fitur utama Makefile adalah \textit{dependency tracking}; ia hanya akan mengeksekusi ulang kompilasi jika timestamp file \textit{dependency} lebih baru daripada file
\textit{target}, menghemat banyak waktu pada proses \textit{development} \cite{gnu_make_manual, gcc_overall_options}.

\section{Pembahasan}
\label{sec:pembahasan}
Berdasarkan serangkaian percobaan yang telah dilakukan, proses pembangunan perangkat lunak terbukti membutuhkan langkah-langkah yang terstruktur, dimulai dari kompilasi tingkat
dasar hingga otomatisasi. Pada bagian Sistem Build dengan GCC (merujuk pada Bagian \ref{subsec:langkah-gcc}), terlihat jelas bahwa \textit{source code} bahasa tingkat tinggi
(seperti \texttt{perkenalan.c}) tidak dapat langsung dieksekusi oleh sistem operasi. Melalui perintah \texttt{gcc} dan tambahan \textit{flag} spesifik (\texttt{-E}, \texttt{-S},
\texttt{-c}), setiap tahapan kompilasi dapat diobservasi secara mandiri. Mulai dari ekspansi \textit{macro} dan \textit{header} pada tahap \textit{preprocessing}, penerjemahan instruksi
menjadi representasi \textit{assembly}, konversi menjadi file objek biner, hingga penggabungan file objek dengan library standar pada tahap \textit{linking}. Analisis kompilasi manual
ini memperlihatkan transparansi bagaimana compiler menerjemahkan logika pemrogram ke bahasa mesin.

Namun, ketika masuk ke tahap kompilasi yang lebih kompleks atau melibatkan manipulasi file berulang, melakukan eksekusi perintah GCC secara manual menjadi kurang efisien. Hal ini dijawab
pada bagian Penulisan Makefile Mendasar (merujuk pada Bagian \ref{subsec:langkah-makefile-dasar}). Terlihat bahwa pembuatan \texttt{Makefile} menyederhanakan interaksi pengguna dengan compiler.
File \texttt{Makefile} bertindak sebagai aturan (\textit{rule}) yang menghubungkan target hasil akhir dengan prasyarat file utamanya (\textit{dependencies}). Dengan memanggil perintah \texttt{make},
seluruh prosedur pemanggilan \texttt{gcc} dilakukan oleh sistem berdasarkan konfigurasi di dalam file tersebut. Kendala sintaks seperti penggunaan karakter \texttt{TAB} pada resep juga menjadi
pembelajaran penting untuk menghindari \textit{error missing separator} yang sangat sering terjadi saat mendefinisikan \textit{recipe} di Makefile.

Otomatisasi tersebut menjadi semakin kokoh saat menangani proyek yang bersifat modular, seperti pada implementasi fungsi matematika dan kalkulator (merujuk pada Bagian \ref{subsec:langkah-makefile-kompleks}
dan Bagian \ref{subsec:langkah-kalkulator}). Pada kasus ini, kode dibagi menjadi komponen antarmuka program (\textit{header}), implementasi fungsi terpisah (\textit{source file}), dan logika pemanggilan utama
(\textit{main}). Dalam hal ini, fitur variabel (\textit{compiler flags} seperti \texttt{CC=gcc}) dan variabel otomatis seperti \verb|$@| (merujuk pada target) dan \verb|$^| (merujuk pada dependensi)
membuat penulisan \texttt{Makefile} jauh lebih ringkas dan dinamis. Proses \textit{dependency tracking} yang dimiliki GNU Make juga terbukti bekerja efisien, memastikan hanya file yang mengalami modifikasi
yang akan dikompilasi ulang, sekaligus menyediakan target praktis seperti \texttt{make clean} untuk manajemen ruang kerja (\textit{workspace}) yang bersih.

\section{Kesimpulan}
\label{sec:kesimpulan}

Berdasarkan hasil praktikum yang telah dilakukan, dapat disimpulkan bahwa proses kompilasi kode sumber C/C++ menggunakan \textit{GNU Compiler Collection} (\texttt{GCC}) telah dipahami melalui praktik langsung.
Proses kompilasi tidak berlangsung secara tunggal, melainkan melalui beberapa tahapan yang saling berurutan.

Pada praktikum ini juga dipahami bahwa tahapan \textit{build} terdiri dari \textit{preprocessing}, \textit{compiling}, \textit{assembling}, dan \textit{linking}.
Setiap tahapan tersebut memiliki fungsi masing-masing dalam mengubah kode sumber menjadi program yang dapat dieksekusi.

Selain itu, konsep dasar otomatisasi kompilasi (\textit{build automation}) perangkat lunak telah dipahami. Konsep ini memungkinkan proses kompilasi dilakukan secara otomatis sehingga tidak perlu menjalankan
perintah \texttt{gcc} secara manual berulang kali ketika terjadi perubahan pada program.

Struktur dasar file \texttt{Makefile} juga telah dipahami, yang terdiri dari \textit{target}, komponen syarat (\textit{dependency}),
dan resep (\textit{recipe}). Struktur ini digunakan untuk mengatur proses \textit{build} agar lebih terstruktur, otomatis, dan mudah dikelola.

Penggunaan variabel kompilator dan \textit{flags} dalam \texttt{Makefile} telah dipraktikkan. Variabel ini digunakan untuk mempermudah pengaturan perintah
kompilasi sehingga proses \textit{build} menjadi lebih fleksibel dan mudah disesuaikan.

Terakhir, penggunaan \textit{automatic variables} seperti \verb|$@| dan \verb|$^| telah dipahami. Variabel ini sangat membantu
dalam menyederhanakan penulisan \texttt{Makefile} karena dapat merepresentasikan target dan dependency secara otomatis tanpa perlu ditulis berulang.
